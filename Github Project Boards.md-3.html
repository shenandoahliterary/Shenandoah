Github Project Boards…

create a platform for you to see and keep up with issues and pull requests in various columns.

Benefits

  • Stay organized, prioritize work and monitor progress on projects
  • Create customized workflows
  • Arrange issues and pull boards, which serve as records to document your work
  • Track release checklists
  • Build road maps

Best Practices

The building blocks, or “cards,” of project boards are issues and pull requests, so to get the most out of GitHub’s project board feature, you’ll want to use these right. Here are some pointers to begin using this feature.

  • Using GitHub issues is an easy way for people to follow the direction of a project. You can file issues, concerning goals, questions, requests, and discussions with your team.

  • There are two types of boards:

    • Repository: Boards for 1 repository
    • Organization: Boards across several repositories

(Which board you use depends on the size and type of project you are working with.)

  • To Create a project board
    • Click on “Create New Project”
    • Add a name for your project and a description
    • Choose from a list of template options
      • None: Start from scratch with a completely blank board. You can add columns and create automation settings yourself
      • Basic Kanban (Kanban is a style of columns): has three columns for “To Do,” “In Progress,” and “Done”
      • Automated Kanban: Has built-in triggers to automatically move issues and pull requests across “To Do,” “In Progress,” and “Done”
      • Automated Kanban with Review: Everything included in the Automated kaban template plus additional triggers for pull request reviews
      • Bug Triage: Triage and prioritize bugs with columns for To Do,” “High Priority,” “Low Priority,” and “Closed.”
    • Click on “Create” and you’ll be presented with a board, consisting of designated columns to manage your project.

What’s Next?

For example, let’s say you chose the “Basic Kanban” template. You should see your three columns: To Do, In Progress, and Done.

  • You can add notes in a column by clicking on the plus sign on the right corner of a column. You can then drag these notecards amongst columns. These can be further manipulated by clicking on the dots on the right side of the card.

  • On the right hand side of the page, there is a space to add a card. This is where you might drag an issue into the “To Do” panel or column.

  • This will be the first location for the issue.

  • The To-Do should be used for your current work. You will want to create a back log to organize your work. Add a new column and call it “Back Log.” Create the column. Drag it to the left of the “To Do” column. Any brand new issue you create will exist in the back log and then what you are working on can exist in the “In Progress” column.

Now a basic Kanban style project board has been created and you should be able to manage and move issues around.

Setting up Automated Workflows allows you to…

automatically move issues through their lifecycle without having to manually input this information, you’ll want to set up automations.

  • For example, if you want to have issues sent to the “To Do” column on their own, you would click on the three dots in the upper right hand column and click “Automation.”
  • Next, select a preset: “None, To do, In Progress, or Done.” You can manage the behavior of issues in this tab accordingly.
  • Last, click “update automation.”"

Now you can integrate your workflow and see issues move through their lifecycles automatically.

Project boards are effective only when a team actively uses it to track the progress of a project.
